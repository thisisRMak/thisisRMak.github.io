What each surface means

Bias²
The error left over because the model class is too simple to represent the true function — the approximation error. Formula: bias²(c) = a·e^(−α(c−1)). It falls as complexity c grows, and it's flat along the n-axis: adding data can't fix a model class that fundamentally can't express the target.
Variance
How much the fitted model swings from one training sample to another — its sensitivity to the particular data it saw. Formula: variance(c, n) = b·c / n. It rises with complexity (flexible models chase noise) and shrinks as n grows (more data steadies the fit).
Bias² + Variance
The reducible error — everything you could remove with a better-chosen model or more data. It's the generalization error with the irreducible noise σ² subtracted off, so it sits just below the green surface.
Training error
Error measured on the exact data the model was fit to. It's optimistic — biased low — because the model has already adapted to those points. Formula: (σ² + bias²)·n/(n+κc). With little data a flexible model nearly interpolates (≈ 0, front-left); as n grows it climbs toward the plateau σ² + bias². The height from here up to the green surface is the generalization gap, which tracks the variance.
Generalization error
Expected error on unseen data drawn from the true distribution — what you actually care about. Formula: σ² + bias²(c) + variance(c, n). This is exactly what a held-out test set estimates, which is why "test error" and "generalization error" name the same quantity. It's the valley whose floor drops — and whose lowest point drifts toward higher complexity — as data grows.

The two flat CS229 plots are slices of this: fix n and cut across complexity to get the U-shaped bias–variance tradeoff; follow the valley floor as n grows to get the learning curve.
